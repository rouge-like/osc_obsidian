# 📄 퍼즐원스튜디오 서류 합격 포인트 및 자소서 분석

## 1. 이력/경력의 압도적 '핏(Fit)'
* **하이퍼 캐주얼 팩토리 경험:** '콘텐츠깎는청년'에서 2~3일 단위 스프린트로 15종 이상의 프로토타입을 찍어낸 경험. 퍼즐원스튜디오 같은 캐주얼/퍼즐 기반 모바일 게임사에서 가장 목말라하는 '압도적인 개발 속도와 실행력'을 정확히 저격한 이력.
* **데이터 기반 사고:** 단순히 게임을 만든 것을 넘어 '국가/연령별 Retention, Playtime 데이터'를 분석해 밸런싱을 조정했다는 성과는 컬처핏 면접에서 보여준 '논리와 데이터 중심의 스탠스'를 서류 단계부터 미리 증명함.

## 2. 자소서 주도 면접 매핑 분석
면접관들은 지원자의 자소서를 읽고, 형광펜을 친 핵심 키워드들을 그대로 질문으로 던졌다. 즉, 자소서에 쓴 4개의 문단이 곧 기술 면접의 목차가 되었다.

### 덫 1: 엔진 비의존성과 수학적 로직 (Drawing Roller)
> **📄 자소서 원본 참조 (블록 링크)**
> ![[퍼즐원스튜디오_이력서#💡 엔진 비의존성과 수학적 로직 (Drawing Roller)]]

* **실제 기술면접 질문:** "Stencil buffer로 최적화 어떻게 했는지?", "Verlet 적분법에서 코드로 옮기는 게 어려웠는지?"
* **분석:** 모바일 캐주얼 게임사에서 가장 좋아하는 '가벼운 수학적 하드코딩' 역량을 어필. 면접관 입장에서는 무조건 물어볼 수밖에 없는 매력적인 최적화 키워드.

### 덫 2: 아키텍처와 유지보수성 (SO Event Pattern)
> **📄 자소서 원본 참조 (블록 링크)**
> ![[퍼즐원스튜디오_이력서#💡 아키텍처와 유지보수성 (SO Event Pattern)]]

* **실제 기술면접 질문:** "SO base event 패턴의 장단점과 싱글톤과의 비교?"
* **분석:** 주니어들이 남발하는 싱글톤을 비판적으로 바라보고, SO 기반 결합도 해제를 논한다는 것 자체가 시니어급의 아키텍처 고민임. "우리 회사의 글로벌 타이틀 20개 유지보수에 기여하겠다"는 명분까지 완벽함.

### 덫 3: 모바일 최적화 (Anatomy VR)
> **📄 자소서 원본 참조 (블록 링크)**
> ![[퍼즐원스튜디오_이력서#💡 모바일 최적화 (Anatomy VR)]]

* **실제 기술면접 질문:** "오브젝트 풀링에서 순간 spike 해결법?", "GC spike 찾는 과정 및 메모리 프로파일링 방법?"
* **분석:** 'Zero-Allocation'과 'GC 스파이크'라는 정확한 최적화 용어를 사용함으로써, 면접관이 프로파일링과 메모리 관리 역량을 묻도록 성공적으로 유도함.

### 덫 4: 개발 파이프라인 자동화 (AI & MCP)
> **📄 자소서 원본 참조 (블록 링크)**
> ![[퍼즐원스튜디오_이력서#💡 개발 파이프라인 자동화 (AI & MCP)]]

* **실제 컬처핏/기술면접 질문:** "MCP 활용 사례?", "AI 관련 비전공자에게 설명 부탁", "AI 활용 풀 사이클?"
* **분석:** 퍼즐원스튜디오의 최대 관심사인 'AI 연동'을 완벽하게 저격한 문단. 이 문단 하나로 컬처핏 면접의 절반 이상의 시간을 본인이 주도하는 AI 기술 토크로 이끌어감.

## 3. 최종 인사이트 (Insight)
* **자소서는 면접관을 조종하는 리모컨이다:** 면접에서 방어하기 힘들거나 자신 없는 기술은 자소서에서 과감히 덜어내고, 내가 면접장에서 10분 이상 떠들 수 있는 강력한 무기(Verlet 적분, SO 패턴, RAG/MCP)만 전략적으로 배치해야 한다.
* **엔지니어의 자소서는 '키워드' 싸움이다:** '열심히 했습니다'가 아니라 'Zero-Allocation', 'Stencil Buffer', 'Model Context Protocol'처럼 면접관의 눈에 확 꽂히는 구체적이고 전문적인 기술 키워드를 썼기에 서류 필터링을 뚫고 퀄리티 높은 면접 질문을 유도해 낼 수 있었다.
