# 🏢 플레이하드 코딩테스트 복기

## 1. 테스트 개요 (Overview)
* **제한 시간:** 150분
* **문제 유형:** 알고리즘 2문항

## 2. 알고리즘 풀이 로그 (Data Log)

### 🪜 [문제 1] 사다리 타기 문제
* **요구사항:** 사다리 시작점 개수 N과 가로 연결 선분 `int[2]` 배열이 주어졌을 때, 각 시작점에서 출발한 최종 도착지점 배열 구하기.
    * *예시:* N = 3, 연결 = [[0,1], [0,2]] -> answer = [1, 2, 0]
* **나의 접근법 (바텀업 스왑 최적화):**
    * 초기 `answer` 배열을 `[0, 1, 2 ... N-1]`로 초기화한다.
    * 사다리의 **맨 아래에서부터 역순(Bottom-up)**으로 가로 선분을 순회한다.
    * 연결된 두 지점 a, b를 만날 때마다 `answer[a]`와 `answer[b]`의 값을 스왑(Swap)한다.
* **디버깅 & 최적화 포인트:**
    * 각 시작점마다 위에서 아래로 전부 탐색하는 이중 for문(Top-down)을 사용하면 O(N^2) 이상의 무거운 시간 복잡도가 발생한다.
    * 하지만 맨 아래부터 역으로 올라오며 스왑하는 방식을 쓰면, 가로 선분의 개수만큼만 순회하는 O(N) 수준으로 탐색을 끝낼 수 있는 아주 효율적인 로직을 구현해 냄.

### 👁️ [문제 2] 던전 시야(Vision) 확보 문제
* **요구사항:** 장애물(true/false)이 배치된 2D 그리드 던전에서, 플레이어 스폰 위치를 기준으로 도달 가능한 영역 + 그 영역의 상하좌우(장애물 포함)까지 시야가 열림. 
    1) 기본 상태에서 열리는 시야의 개수 구하기.
    2) 특정 위치의 장애물을 1개 파괴했을 때 새롭게 열리는 전체 시야 개수 구하기.
* **나의 접근법 (BFS 응용):**
    * **BFS(너비 우선 탐색)** 알고리즘을 사용하여 도달 가능한 빈 공간을 탐색한다.
    * 핵심 로직: 탐색 도중 다음 칸이 '장애물'일 경우, Queue에 추가(Enqueue)하지 않는다. 대신 해당 장애물 칸의 `visited = true` 처리만 해주고 시야 `count++`를 올려준다.
* **디버깅 & 최적화 포인트:**
    * 전형적인 플러드 필(Flood Fill) 문제에 '인접한 장애물까지만 카운트'라는 조건이 붙은 문제.
    * `visited` 배열을 활용해 이미 시야가 밝혀진 장애물을 중복으로 카운트하지 않도록 예외 처리(Exception Handling)를 확실하게 걸어둔 점이 핵심.
